Point any OpenAI SDK at it. Get Kilo/opencode routing transparently. Real token-by-token SSE via ACP's message.part.delta.
[try it live] [github]

install

npx acptoapi                      # defaults :4800, kilo :4780, opencode :4790
npx acptoapi --port 8080          # custom port

Requires an ACP backend running: kilo serve --port 4780 or opencode serve --port 4790.

drop-in openai sdk

import OpenAI from 'openai';
const client = new OpenAI({ baseURL: 'http://localhost:4800/v1', apiKey: 'none' });

const stream = await client.chat.completions.create({
  model: 'kilo/x-ai/grok-code-fast-1:optimized:free',
  messages: [{ role: 'user', content: 'hi' }],
  stream: true,
});
for await (const c of stream) process.stdout.write(c.choices[0]?.delta?.content || '');

how it works

    
    
      
  • messages[] → ACP text part (roles flattened, system hoisted)
    • 
      
  • ACP message.part.delta → OpenAI delta.content / delta.reasoning_content
    • 
      
  • partID → part.type lookup from preceding message.part.updated → routes reasoning deltas correctly
    • 
      
  • ACP tool part state → OpenAI tool_calls (input snapshots)
    • 
      
  • session.idle / session.turn.close → finish_reason: stop (no timeout heuristics)
    • 
    

    
True streaming: first chunk arrives in ~1ms after backend responds, one SSE frame per token.
