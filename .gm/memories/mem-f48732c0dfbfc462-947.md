gm-method: a code-duplication audit's highest-yield witness is finding the SAME data-shape struct + same mutator bodies across modules, especially when a comment self-admits the copy (e.g. 'Steps mirror lib/other.js'). A backoff/failure-state primitive {ok,failCount,nextCheck,lastFailedAt}+markFailed/markOk/peekStatus recurring 2-3x across sibling modules is the canonical extract-shared-primitive target (>3 lines, used Nx). Distinguish a near-identical pair (factor into one helper) from a same-concept-different-shape third (leave it). For downstream-compat audits: classify each consumer as (a) direct SDK import with version constraint, (b) HTTP-only no-dep, or (c) transitive-via-another-consumer no-dep -- only (a) carries a semver breakage surface; (b)/(c) only depend on the wire/contract surface staying stable. A vendored browser-SDK bundle shipped by a downstream is a drift vector resolved by a refresh script, not hand-forked code.
