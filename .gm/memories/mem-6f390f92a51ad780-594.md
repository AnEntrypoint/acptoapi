{"key":"acptoapi-app-shell-css-specificity","value":"acptoapi docs app-shell.css specificity trap: '.app-main a' (0,1,2: class+element descendant) outweighs '.btn-primary' (0,1,0), so generic anchor color overrides button colors -- 'Try it live' rendered mint-on-mint invisible. Witness: getComputedStyle(element).color on the docs server. Fix: scope the descendant anchor rule with :not() exclusions -- .app-main a:not(.btn):not(.btn-primary):not(.btn-ghost) { color: var(--panel-accent); }. Root cause was selector specificity, never CSS-variable resolution (--panel-accent-fg was correct)."}
